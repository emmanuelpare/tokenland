# TL;DR
- Ornith-1.0 propose des modèles de 9B à 397B paramètres, post‑entraînés sur Gemma 4 et Qwen 3.5, spécialisés pour le codage agentique.
- Leur cadre d’auto‑apprentissage utilise le renforcement pour générer non seulement les solutions de code, mais aussi le scaffolding qui les entoure, améliorant ainsi la qualité globale.
- Disponibles sous licence MIT, ils surpassent les modèles open‑source précédents sur des benchmarks tels que SWE‑Bench, Terminal‑Bench et NL2Repo.

# Ce qui se passe
Ornith-1.0, annoncé le 29 juin 2026 par l’équipe DeepReinforce, est une famille de modèles de langage conçue explícitement pour les scénarios de **codage agentique** (où un modèle génère du code, exécute des actions, et itère). Les variantes disponibles sont :
- **Ornith-1.0‑9B** (densément connecté)
- **Ornith-1.0‑31B‑MoE** et **‑35B‑MoE** (mixture‑of‑experts)
- **Ornith-1.0‑397B‑MoE** (le plus grand modèle)

Tous sont post‑entraînés à partir de **Gemma 4** et **Qwen 3.5**, puis affinés grâce à un cadre d’auto‑amélioration basé sur l’apprentissage par renforcement (RL). Au lieu de produire uniquement le texte de réponse, le modèle apprend à générer à la fois le **code cible** et le **scaffolding** (commentaires, structures de tests, définitions d’interfaces) qui l’entoure, ce qui permet au système d’itérer plus efficacement sur des tâches complexes.

Les résultats montrent des améliorations substantielles :
- Sur **SWE‑Bench Verified**, Ornith‑1.0‑397B atteint **76,4 %**, dépassant Qwen 3.5‑35B (70 %) et Gemma 4‑31B (52 %).
- Sur **Terminal‑Bench 2.1**, le modèle 397B obtient **64,2 %** (vs 41,4 % pour Qwen 3.5‑35B).
- Sur **SWE‑Bench Multilingual**, il atteint **69,3 %**, bien au‑dessus des baselines ouvertes existantes.

Le code source et les poids sont publiés sous licence **MIT** sur GitHub (deepreinforce‑ai/Ornith‑1), rendant le modèle totalement libre d’utilisation, de modification et de déploiement sans restriction géographique.

# Pourquoi ça compte pour un dev
Pour les ingénieurs qui construisent des agents de code ou qui utilisent des copilotes d’IA, Ornith‑1.0 offre plusieurs avantages concrets :
1. **Auto‑amélioration intégrée** : le modèle apprend à produire non seulement le bon algorithme, mais aussi le contexte de test et de documentation qui réduit les cycles d’itération manuelle.
2. **Performance de pointe** : les scores supérieurs sur les benchmarks de programmation signifient moins de faux positifs et une meilleure capacité à gérer des projets réels dès la première génération.
3. **Ouverture totale** : licence MIT, pas de frais, pas de limites d’usage – idéal pour les startups, les équipes de recherche ou les environnements soumis à des contraintes de conformité.
4. **Taille modulable** : selon le budget de calcul, on peut choisir le 9B pour du prototypage rapide ou le 397B pour des tâches exigeantes sans dépendre d’un fournisseur propriétaire.

# Ce qu’en dit la communauté
Sur Hacker News, l’annonce d’Ornith‑1.0 a rapidement dépassé **101 points** avec de nombreux commentaires soulignant l’originalité de l’approche « self‑scaffolding » et la pertinence pour le développement d’agents autonomes. Bien que les discussions sur Reddit/r/LocalLLaMA soient encore limitées à ce stade, les premiers retours soulignent la clarté du README et la facilité de lancement grâce aux scripts fournis.

# À retenir / verdict
**Vaut absolument le coup d’œil** pour tout développeur intéressé par l’avenir du codage assisté par IA, en particulier ceux qui travaillent sur des agents capables de boucler sur leur propre sortie. La combinaison de performances élevées, d’une licence permissive et d’un mécanisme d’auto‑amélioration place Ornith‑1.0 parmi les références actuelles du domaine.
